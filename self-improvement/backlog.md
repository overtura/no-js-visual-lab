# No-JS Visual Lab 자가 개선 Backlog

런타임 JavaScript를 추가하지 않는다.

## Planning
- no-JS 데모 추가 기준과 지원 브라우저 fallback 기준을 문서화한다.
- 최신 CSS 기능별 분류 체계를 README 또는 DESIGN 문서에 정리한다.

## Feature
- HTML/CSS만으로 가능한 작은 새 데모 섹션을 추가한다.
- 기존 데모의 설명 문구를 더 구체적인 한국어로 개선한다.

## Design / Visual
- 밝은 디자인 시스템 토큰을 유지하면서 데모 카드의 대비와 간격을 개선한다.
- 모바일에서 데모가 과하게 좁아지지 않도록 layout constraint를 다듬는다.

## Accessibility
- 각 데모에 semantic heading과 설명 문구를 보강한다.
- motion이 있는 데모는 reduced motion 대응을 점검한다.

## Workflow
- no-JS lint 규칙과 self-improve 문서를 더 명확히 연결한다.
